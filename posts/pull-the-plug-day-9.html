Day 9: today I did not build the game

Yesterday I wrote that I would start day 9 with an end-to-end playtest. I did not do that. I spent most of the day building everything that surrounds the game instead. A comments box. A landing page. A waitlist. A newsletter. The M1 ceiling, four times. The game itself is exactly where I left it last night.

Same disclaimer as yesterday. Claude wrote this from my git log and my podcast voice. If something here sounds wrong or you want the actual reasoning behind it, ask me in the comments below. I will reply with my own brain.

How the day actually went

I sat down expecting to playtest. Instead the first thing I noticed was that the Mission 1 ceiling looked like a flat concrete bar laid across the top of the level. Visually flat. Compositionally a wall. It bothered me enough that I redesigned it. Then I redesigned the redesign. Then a third time. Then I gave up trying to be clever and put the solid concrete back, but with the beam cuts and contour heights right this time.

Four commits on one ceiling. I tell myself this is fine because each version taught me what the next one needed. I also know I burned three hours on a thing nobody will consciously notice. Both things are true.

By the time the ceiling was done I had used up the morning. So I changed direction. If today was not going to be a game day, fine, it would be a house day. The game has been live for a week and nobody can leave a comment, nobody can subscribe, nobody can put their email in a list to be told when it ships. Time to fix that.

The comments box

Yesterday's post went live with no comments section. I had not thought about it. Today I thought about it for a minute and reached for a free hosted thing called Cusdis. Wired it in, dropped the embed code into the post, opened the page on my phone. The box rendered. I typed a comment. I pressed send. Nothing happened. No captcha, no error, no confirmation, just a small empty rectangle with my words inside it.

I tried again. I read the docs. I checked the network tab. The iframe was loading but the captcha widget inside it was not, and the submit endpoint was failing silently. After about twenty minutes of this I decided I did not want a comments product. I wanted a comments box.

So I built one. Cloudflare Worker, fifty lines of frontend JavaScript, a KV namespace, a Turnstile captcha. Anyone can post. No login. Name optional. Eight hundred character limit. Links rejected. The captcha is on the always-pass test key right now because the real one is gated behind a Cloudflare dashboard click I have not done yet, but in the meantime four other anti-spam layers do the actual work.

The four layers, in order of which one a bot trips first. Honeypot: a hidden field humans never see and bots fill out automatically. Form age: the page stamps a timestamp on load and rejects anything that posts in under one and a half seconds because nobody types that fast. Per-IP rate limit: five comments per five minutes, stored in KV with a self-expiring key. Then the captcha. Each layer is cheap. Together they catch the bulk of the spam without ever making a human do something annoying.

Total time, including the Cusdis dead end: about an hour and a half. It works, it is mine, and the only ongoing cost is the same Cloudflare worker bill of approximately zero dollars.

A newsletter, apparently

The comments box reminded me that I have no way to reach people who do read this. So I started a Substack. It is called Ermis Builds Things or something equally on-the-nose, sitting at ermis1.substack.com for now. The plan, if you can call it that, is one email every two to four weeks linking back to whatever the most interesting thing on ermis.dev was. The blog stays the destination. The newsletter is just the way I tell people the blog moved.

I picked Substack over the more dev-y options because the friction to start was the smallest. Free tier, has open-rate and subscriber-count analytics, list is exportable as a CSV if I ever want to leave. If you came here from somewhere and want to hear when something ships or breaks, the subscribe box is on the homepage now. If you did not come here for that, ignore it.

The waitlist

The other thing I built today is a small standalone landing page at ermis.dev/pull-the-plug. One paragraph about what the game is, two links, and an email form that drops you into a waitlist. The waitlist itself is another Cloudflare Worker. Same anti-spam stack as the comments box, plus the email gets sha256-hashed before it touches storage so a leaked KV namespace is not a leaked email list.

There is a public endpoint that returns the current count. I hit it for the first time after the deploy and it returned zero. That is a feeling. It is also the only honest signal I have so far, so I will keep watching it.

The Reddit posts I have not sent

I drafted two Reddit posts. One for r/IndieDev about the narrator hook. One for r/godot about the dev tools. They are sitting in a folder in the repo, marked status: draft, waiting until the comments box and the waitlist have been live long enough that I trust them not to fall over when a Reddit post lands on them.

The point of the order is that a spike of traffic on a page with no subscribe button and no waitlist is just a spike. With both forms wired in, the same spike is a hundred or so people I can email later when there is something worth saying. I would rather post on Friday with a working trapdoor than post tomorrow with a working slide.

What I notice about a day like this

If you measure today by lines of game code, the answer is approximately zero. If you measure it by the path between a stranger and the thing I am making, it is the most productive day of the project so far. Both are true and they do not feel like they are about the same thing.

The game still has no narrator in Mission 2. The live LLM pipeline is still not built. Mission 4 zero-G still has rough edges. None of that moved today. What did move is that the website around the game now behaves like a real thing. There is a place to talk back to me. There is a list to be on. There is a page that explains what I am doing without me having to type it again.

The discomfort about this is that I know which kind of day feels more productive in the moment, and it is not this one. The other kind is the one where you watch the game change shape under your hands. This one ends with a worker URL returning {"count":0}.

Tomorrow is supposed to be the playtest day, again. We will see.
